Week 1: Data Link Layer Framing Methods

3 Programs

Overview

Data Link Layer framing is essential for organizing data transmission between network nodes. This week explores various framing methods that ensure data integrity and proper synchronization between sender and receiver.

Character Counting

Basic

Purpose: Uses a count field to specify the number of characters in the frame.

Advantage: Simple implementation, minimal overhead.

Disadvantage: Vulnerable to count field corruption, poor error recovery.

Use Case: Early network protocols, educational purposes.

Character Stuffing

Intermediate

Purpose: Uses special delimiter characters (STX/ETX) to mark frame boundaries.

Advantage: Better error recovery, frame boundaries clearly marked.

Disadvantage: Need to handle special characters in data (escape sequences).

Use Case: Text-based protocols, legacy systems.

Bit Stuffing

Advanced

Purpose: Uses unique bit patterns as frame delimiters, stuffing bits to prevent false flags.

Advantage: Works with any data type, efficient bandwidth usage.

Disadvantage: Complex implementation, bit-level manipulation required.

Use Case: HDLC, PPP protocols, modern networking.

Learning Objectives

  • Understand the need for frame delimitation in data communication
  • Compare different framing methods and their trade-offs
  • Implement basic framing algorithms in C programming
  • Analyze error scenarios and recovery mechanisms

Week 2: CRC Code Implementation

Error Detection

Overview

Cyclic Redundancy Check (CRC) is a powerful error-detecting code used in digital communication networks. It uses polynomial division to generate check bits that detect transmission errors with high probability.

CRC Algorithm

Intermediate

Purpose: Detects errors in data transmission using polynomial mathematics.

Advantage: High error detection rate, standardized polynomials available.

Disadvantage: More complex than simple checksum methods.

Use Case: Ethernet frames, WiFi protocols, storage devices.

Learning Objectives

  • Understand polynomial division in error detection
  • Implement CRC-16 algorithm in C programming
  • Analyze CRC error detection capabilities
  • Compare CRC with other error detection methods

Week 3: Sliding Window Protocol (Go-Back-N)

Flow Control

Overview

Sliding Window Protocol allows multiple frames to be transmitted before receiving acknowledgment. Go-Back-N is a specific implementation that ensures proper sequencing and flow control.

Go-Back-N Protocol

Advanced

Purpose: Efficient data transmission with error recovery mechanism.

Advantage: Pipelined transmission, good throughput efficiency.

Disadvantage: May retransmit unnecessary frames on errors.

Use Case: TCP protocol, reliable data transfer protocols.

Learning Objectives

  • Understand window-based flow control mechanisms
  • Implement Go-Back-N protocol simulation
  • Calculate protocol efficiency and throughput
  • Analyze error recovery strategies

Week 6: Distance Vector Routing Algorithm

Dynamic Routing

Overview

Distance Vector Routing is a dynamic routing algorithm where each router maintains distance vectors and exchanges them with neighbors. It's the foundation of protocols like RIP.

Bellman-Ford Algorithm

Advanced

Purpose: Distributed routing with automatic route discovery.

Advantage: Simple implementation, automatic convergence.

Disadvantage: Count-to-infinity problem, slow convergence.

Use Case: RIP protocol, small to medium networks.

Learning Objectives

  • Understand distributed routing algorithms
  • Implement distance vector protocol simulation
  • Analyze convergence properties and limitations
  • Study routing loop prevention techniques
